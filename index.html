Pushbag rules

You put one NIS coin in the jar if you are guilty of either:

  1. Hurtful Push

    • You committed code that results in some build error.
    • Your committed code fails a test.
    • Your changes prevent someone else from working.

  2. Ridiculous Bug

    • You wrote ridiculously invalid HTML or CSS.
    • Your caused a page to trigger a Javascript error on page load.
    • You have a syntax error or you broke the coding guidelines.

  3. No push

    You did not push any code to a remote repository all day.

  4. Bad deploy

    You deployed bad or buggy code to an external server.

  5. Bad merge

    You did not merge your changes properly, and committed conflicted code or overwritten somebody else's work.

No excuses!

  1. Put a Shekel in the jar.
